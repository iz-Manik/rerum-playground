About RERUM Playground

RERUM Playground is designed to act as a self-guided interactive tutorial for tools involved in working with digital objects.

Here, you can explore tools for adding annotations, geolocation data, and many other enhancements to JSON objects without worrying that you may break something.

What is a JSON object?

JSON (JavaScript Object Notation) is a simple way to store and send information between computers. Think of it like a person's case file, but for data. JSON organizes information using curly braces {}, square brackets [], and colons :. It's easy for both humans and computers to read.

For example, a person's details in JSON might look like this:

{

"name": "Jane Doe",

"age": 28,

"jobTitle": Software Engineer

}

JSON is used widely on the internet to send data between a server (like a website's backend) and a user's browser.

What is JSON-LD?

JSON-LD (JSON for Linked Data) is like a more savvier JSON. It does everything JSON does, but adds context to the data. This context helps computers understand the meaning of the data, not just its structure.

For instance, when JSON-LD sees "name," it can understand that it's talking about a person's name, not a pet's name or a company's name. This makes it easier for different systems to work together and understand each other's data.

JSON-LD has an element called @context, which is like a dictionary that explains what the terms in your data mean. It links your data to standardized definitions on the internet. For example,

{

"@context": "https://www.com",

"name": "Jane Doe",

"jobTitle": "Software Developer"

}

Here, "@context": "https://www.com" tells computers to look up the meanings of "name" and "jobTitle" on www.com. This helps ensure that different systems interpret the data in the same way.

What is a JSON Manifest?

A manifest provides information about a web application. It works like a cargo manifest for IIIF objects, containing links to all the resources which make up the object.

Manifests are created so the IIIF objects that they represent can be opened and explored in viewers or other tools.

For more information on manifests, please visit:

https://iiif.io/guides/using_iiif_resources/
